		\subsection{Product Functions}
		The NavUP system will help visitors to our campus as well as employees and students to get from one venue to another venue with ease. In summary these are the following things that the NavUP system will consists of:
			\begin{itemize}
			\item locate the current location of the user.
			\item finding the route of destination place.
			\item search for a place.
			\item view the map.
			\item logging and registering of the user.
			\item manage users.
			\item keep track of the users
		\end{itemize}

		\subsection{User Charateristics}
			\begin{itemize}	
			\item User must be able to use smartphones applications and comfortable with them
			\item	User must atleast be able to read and write and must have completed atleast grade 10
			\item	User must have experience with smart devices and be able to navigate through most applications of smart devices
			\item	User must be familiar with the latest technologies like google maps, wifi
		\end{itemize}

		\subsection{Constraints}
		\begin{itemize}
		\item	The platform must exist in two medium namely android and iOS application
		\item The system will use the universityâ€™s wifi access points
		\item GPS will be used to get the route details and directions
		\item The system will operate within the main campus of the university
		\item Database will be used to store different type of information like activities, events and user details
	\end{itemize}

		\subsection{Assumptions and Dependencies}
			\begin{itemize}
				
			\item	It is assumed that the user is within the hatfield campus
			\item	It is assumed that the user owns a smart device
			\item	It is assumed that wifi connection is available
		\end{itemize}

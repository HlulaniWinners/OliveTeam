	\section{Overall Description}
		\paragraph\indent
		
		\subsection{Product Perspective}
			\paragraph\indent
				The software will be a stand-alone mobile navigation application built for both Android and iOS devices.\\
				
				It differs from related products in the marketplace as it focuses on both indoor and outdoor navigation of the user within the grounds of a single property, instead of the usual street navigation.\\
				
				This system can be extended to integrate with other applications that can notify a user about activities/interests close to the user.
			
				\subsubsection{System Interfaces}
					\paragraph\indent
				External interfaces will include: 
				
				\begin{itemize}
				\item WiFi access points
				\item Google maps API
				\item GPS
				\item Event calendar applications
				\end{itemize}

				\subsubsection{User Interfaces}
					\paragraph\indent
				The user will interface with the application using the Android or iOS mobile applications. These applications will give the user access to the following pages:
				
				\begin{itemize}
				\item Registration/login pages
				\item Map screen
				\item Notifications page
				\item Settings page
				\end{itemize}
				
				Additionally, an admin user will have access to the following pages:
				
				\begin{itemize}
				\item Admin page
				\item Create notifications page
				\end{itemize}

				\subsubsection{Hardware Interfaces}
					\paragraph\indent
				The application does not have any direct hardware interface requirements.

				\subsubsection{Software Interfaces}
					\paragraph\indent
				The application will determine the user's location using a combination of the Google Maps API, the device's GPS system, and nearby WiFi access points. \\
				
				It will also integrate with other applications handling upcoming events/activities/interests and notify the user about relevant and nearby ones.

				\subsubsection{Communications Interfaces}
					\paragraph\indent
				Communications will be encrypted using HTTPS protocol.	

				\subsubsection{Memory}
					\paragraph\indent
				As this will be a mobile application for students, memory constraints need to be below the average RAM of a student's smart phone. Earlier devices had a per-app cap of 16MB, and later devices increased that to 24MB or 32MB. Therefore in order to make the application accessible to the largest possible market it should preferable not exceed 16MB of RAM, and definitely not 24MB of RAM.
